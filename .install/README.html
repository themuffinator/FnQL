1. Introduction

FnQL is an engine project for retail Quake Live compatibility. It uses FnQ3 as the imported technology baseline and QLSRP as the reconstruction reference.

The goal is not to rebuild Quake Live game code. FnQL should load and serve retail-compatible content and module boundaries, preserve Quake Live engine behavior, and carry FnQ3's renderer, audio, platform, and tooling improvements forward where they remain compatible.

2. Scope

FnQL is scoped as a modern Quake Live engine, with retail assets and game modules remaining external.

In scope

  • Retail Quake Live Steam installation compatibility, including practical default-path discovery and baseq3 data expectations.
  • Engine compatibility reconstructed from QLSRP evidence: filesystem behavior, pak/pk3 loading, protocol and demo paths, VM/native module ABI, server and client glue, renderer data formats, platform services, and runtime identity.
  • A project-owned fnql-web.pak settings overlay that exposes FnQL engine controls and removes unsupported retail post-processing controls while leaving all untouched launcher resources in the legitimate retail web.pak.
  • Modernized engine features inherited from FnQ3, including SDL3, OpenAL, exactly three renderers (GLx, Vulkan raster vk, and ray-traced rtx), console and screenshot improvements, opt-in motion blur, enhanced map lens flares and liquids, optional per-map global fog, and deterministic release tooling.
  • Compatibility validation against a legitimate retail Quake Live installation.

Out of scope

  • Reconstructing or shipping Quake Live qagame, cgame, or UI game code in FnQL.
  • Bundling retail Quake Live assets.
  • Claiming live Quake Live online-service replacement before an explicit, documented, default-off compatibility path exists.

3. Development Status

FnQL is at the initial migration stage.

  • FnQ3 has been imported and rebranded to FnQL.
  • The canonical project identity now targets retail Quake Live on Steam.
  • Maintainer docs identify QLSRP and FnQ3 as the two primary reference repositories.
  • The next development phase is subsystem comparison against QLSRP, followed by focused compatibility reconstruction in modern C++.

Keep fnql-web.pak beside the FnQL executable. It is a sparse settings override, not a replacement for the retail web.pak; the Windows x86 WebUI still requires a legitimate Quake Live installation.

Official Windows packages include the compiled closed-source fnql_steam.dll provider. Its source and Valve's steam_api.dll are not included; the provider loads the latter from the user's legitimate retail installation. See docs/fnql/STEAM_PROVIDER_BINARY_NOTICE.txt.

For build commands, use the Build Guide.

4. Reference Workflow

Use QLSRP before making compatibility claims.

  1. Locate the owning subsystem in the QLSRP reference checkout.
  2. Compare FnQL's inherited FnQ3 implementation against QLSRP source and reference evidence.
  3. Record observed behavior separately from inferred intent.
  4. Reconstruct the engine-side compatibility needed by retail Quake Live.
  5. Validate against a legitimate Steam install of Quake Live.

5. Documentation

The main guides are split between the current build and maintainer references, plus inherited FnQ3 feature docs that will be updated as Quake Live work lands.

Inherited feature guides

  • Display Guide for renderer choice, video modes, render scaling, HDR, bloom, motion blur, map lens flares, enhanced liquids, optional global fog, dynamic lighting, shadow maps, and cel shading.
  • Visuals Guide for player highlighting and other presentation controls.
  • Aspect Handling Guide for menu and cinematic layout options.
  • Audio Guide for backend selection, devices, HRTF, fallback behavior, and troubleshooting.
  • Console Guide for console layout, scaling, completion, and interaction.
  • Screenshot Guide for capture commands, naming, metadata sidecars, watermarks, and cube-map export.
  • Release History for published changelogs and downloads.

Technical and build docs

  • Build Guide for compiling FnQL locally.
  • GLx Renderer Guide for the default OpenGL-lineage renderer, diagnostics, and troubleshooting.
  • RTX Renderer Contract for the hybrid renderer, conservative defaults, capability gates, and retail-safe smoke procedure.
  • Modern Audio Engine Notes for engine architecture and compatibility boundaries.
  • Technical Notes for repository structure, release flow, and maintainer conventions.
