# GLx Feature Closure Matrix

## Status

Accepted current-state matrix, 2026-05-09.

This document is the checked-in feature-closure ledger for GLx. It describes the renderer against the final contract in [GLX_FINAL_CONTRACT.md](GLX_FINAL_CONTRACT.md). GLx has replaced the older OpenGL renderer names; references to the retired renderer2 implementation below are historical feature-provenance notes, not selectable backends. Rows marked `covered` still need to remain covered by blocking runtime proof.

## Status Legend

| Status | Meaning |
|---|---|
| covered | GLx has an implemented path plus current diagnostics, logic tests, or sweep coverage that can keep the feature visible during RC review. |
| partially covered | GLx can exercise some or all of the feature through a shared compatibility path, a gated path, diagnostics-only proof, or a subset of tiers, but promotion still needs ownership or runtime evidence. |
| missing | GLx does not yet have the required GLx-owned implementation or proof path. |

Rows must use exactly one of `covered`, `partially covered`, or `missing`. Empty, unknown, TBD, or mixed status rows are not allowed.

## Promotion Interpretation

The matrix is intentionally conservative. A row can be `covered` for current tracking while its runtime proof remains gated; changes still require the [Consolidation And Regression Rules](GLX_FINAL_CONTRACT.md#consolidation-and-regression-rules), blocking runtime artifacts, and the ownership-proof profile. A row marked `missing` is not ambiguous. It is a named enhancement or historical feature gap that must be implemented or deliberately excluded from FnQL's retail-QL compatibility scope with maintainer approval; it cannot be hidden behind an undeclared renderer alias.
